    CornChat - How it Works

    CornChat runs as a JavaScript bundle injected into your HipChat UI. A continuous loop watches for new HipChat DOM elements, decorates them with markdown and injects a custom toolbar.

    When you tag a message, CornChat posts the tag name and associated HipChat message UUID to a private AWS AppSync-managed GraphQL instance using a GraphQL mutation.

    CornChat uses MQTT over a WebSocket to subscribe to AWS AppSync GraphQL mutations made against tags and reactions. CornChat’s GraphQL schema uses AWS AppSync’s @aws_subscribe directive to wire up the WebSockets on AppSync’s managed GraphQL server. A custom AWS lambda publishes the GraphQL API URL to a public configuration file on S3 so that the CornChat front-end Webpack build can discover it.

    CornChat’s AppSync GraphQL server writes tags, reactions, and HipChat message UUID values to AWS DynamoDB using VTL DynamoDB resolver templates.

    To prevent abuse of the GraphQL server and database, CornChat JavaScript authenticates you by transmitting your HipChat User ID and HipChat OAuth token to a custom lambda that validates your OAuth token against the public HipChat server, https://api.hipchat.com/. Once authenticated, CornChat generates a CornChat API token, tied to your HipChat User ID, which can be used to login to CornChat itself. CornChat persists the CornChat API token in your browser local storage. When CornChat starts up, it invokes a login lambda to log you in with your API token. The lambda generates temporary federated AWS Cognito credentials with permissions to invoke CornChat GraphQL API endpoints.

    CornChat does not store your HipChat OAuth token. It stores your HipChat UserID in a DynamoDB table to associate the user with a CornChat API token, and in another DynamoDB table to associate you with your reaction votes.

    The CornChat UI makes heavy use of React where possible. It separates persistence functionality from UI functionality by using container components and presentation components. The main app component uses the Render Props pattern to explicitly compose all the container and presentation components together.
